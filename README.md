# -HX711-Pressure-Scale-Based-on-HX711-Pressure-Sensor-and-Raspberry-Pi-
这是一份基于树莓派（Raspberry Pi）的智能高精度称重与物联网数据记录系统。该项目不仅仅是一个简单的电子秤，它结合了硬件驱动开发、嵌入式图形界面（GUI）设计以及物联网（FTP）通信技术，形成了一个完整的“感知-交互-传输”闭环。
1. 核心硬件驱动：HX711 与 传感器
这是系统的“感官”部分，负责将物理重量转化为数字信号。
底层控制：代码中自定义了 HX711 类，不依赖外部库，直接通过 GPIO 模拟时序 控制芯片。
双模滤波算法：
   快速模式 (get_weight_fast)：使用中值滤波（采样5次取中位数），反应极快，用于实时显示，能有效过滤偶然的剧烈抖动。
   精准模式 (read_average)：使用均值滤波（采样10-20次），用于去皮和校准，确保基准数据的准确性。
校准逻辑：
   去皮 (Tare)：记录当前零点偏移量 offset。
   校准 (Calibrate)：利用已知砝码计算 scale_ratio，将原始 ADC 数值转换为克（g）。
2. 人机交互：OLED 图形界面 (SSD1306)
这是系统的“面孔”，使用 luma.oled 库驱动屏幕，提供直观的操作反馈。
状态可视化：实时显示重量、系统状态（如“Calibrating...”、“Measuring...”），替代了枯燥的命令行。
智能稳重显示：代码中实现了一套状态机逻辑。只有当重量波动小于 0.5g 且连续 5 次稳定时，才更新屏幕数值。这模拟了商业电子秤的“跳数”效果，避免了用户看到数字乱跳，提升了体验。
容错机制：如果未检测到屏幕，程序会自动降级为仅控制台模式，不影响核心逻辑。
3. 物联网功能：FTP 数据上传
这是项目的“云端”扩展，实现了数据的持久化存储。
协议：使用 FTP (文件传输协议)。
流程：连接服务器 -> 登录认证 -> 切换目录 -> 二进制上传。
目标：将本地生成的record_weight.txt 文件上传至远程服务器（IP: 100.1.1.1）注意：该IP需要修改为你所需要的IP，路径为 /Weight。
应用场景：非常适合实验室数据记录或无人零售场景，数据可自动同步至后台。
4. 交互式控制台
程序提供了一个命令行菜单，用户通过输入指令控制设备：
t (Tare)：归零操作。
c (Calibrate)：校准操作（需输入已知重量）。
w (Weigh)：开始称重，数据自动记录到本地文件。
u (Upload)：将本地记录上传至 FTP 服务器。
q (Quit)：安全退出。
